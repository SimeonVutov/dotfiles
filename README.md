## ‚ö° Features

### ūüĒ∑ Hyprland

> `/.config/hypr/`

Hyprland is the core of this setup ‚ÄĒ a dynamic tiling Wayland compositor with buttery-smooth animations and extensive customization.

- **Tiling & Floating** ‚ÄĒ smart window tiling with flexible per-workspace layout rules
- **Animations** ‚ÄĒ custom bezier curves and animation presets for window open/close, workspace switching, and layer popups
- **Window Rules** ‚ÄĒ per-app floating, opacity, and workspace assignment rules
- **Input** ‚ÄĒ fine-tuned touchpad gestures, mouse sensitivity, and keyboard repeat rates
- **Monitors** ‚ÄĒ multi-monitor support with per-display resolution, refresh rate, and scaling
- **Keybinds** ‚ÄĒ a complete, logically grouped keybind system covering workspaces, windows, apps, media, and scripts
- **Hypridle / Hyprlock** ‚ÄĒ auto screen lock with configurable idle timeout
- **Environment Variables** ‚ÄĒ Wayland-native env vars for proper app compatibility (XDG, cursor, Qt/GTK backends)

### ūüźö Zsh + Oh My Zsh

> `.zshrc`

A highly productive shell environment built on Oh My Zsh with smart quality-of-life improvements.

- **Theme** ‚ÄĒ `agnoster` prompt showing git branch, status, and exit codes at a glance
- **Plugins** ‚ÄĒ `git`, `archlinux`, `zsh-autosuggestions` for inline history-based suggestions
- **Smart Tmux Auto-Attach** ‚ÄĒ on every new shell, automatically attaches to an existing idle tmux session or creates a new numbered one (`session1`, `session2`, ‚Ä¶); prevents orphaned sessions
- **Fastfetch on Login** ‚ÄĒ system info displayed on every new terminal session
- **eza Aliases** ‚ÄĒ `ls`, `ll`, `lt` all use `eza` with icons for a beautiful file listing
- **FZF Integration** ‚ÄĒ `Ctrl+R` launches a fuzzy history search with full preview
- **power-mode Autocompletion** ‚ÄĒ custom Zsh completion function for the `power-mode` script with contextual options (`ultimate`, `balanced`, `-min`, `-max`, `-gov powersave/performance`)
- **Wayland Environment Refresh** ‚ÄĒ `precmd` hook re-exports `WAYLAND_DISPLAY`, `XDG_RUNTIME_DIR`, and friends into each tmux pane so GUI apps never lose their display connection
- **Developer Paths** ‚ÄĒ Raspberry Pi Pico SDK, Picotool, QuestaSim (FPGA/simulation), and pnpm all pre-configured in `$PATH`

### ūüĖ•ÔłŹ Tmux

> `/.config/tmux/`

Terminal multiplexer configured for a smooth, keyboard-driven multi-session workflow.

- **Status Bar** ‚ÄĒ custom styled status line matching the terminal theme
- **Session Naming** ‚ÄĒ automatic sequential session naming (`session1`, `session2`, ‚Ä¶) managed by Zsh
- **Mouse Support** ‚ÄĒ optional mouse for pane resizing and selection
- **Vim-like Pane Navigation** ‚ÄĒ `hjkl` keybinds for moving between panes
- **Copy Mode** ‚ÄĒ vi-style copy mode with clipboard integration via `wl-clipboard`
- **Plugin Support** ‚ÄĒ managed via tpm (Tmux Plugin Manager)
- **Persistent Environment** ‚ÄĒ Wayland display variables refreshed per-pane automatically

---

### ūüé® pywal / wal

> `/.config/wal/templates/`

Automatic theming pipeline that generates a cohesive colorscheme from the current wallpaper and propagates it across the entire system.

- **Templates** ‚ÄĒ custom wal templates for Waybar CSS, Dunst, Kitty, Rofi, and more
- **Hot Reload** ‚ÄĒ changing wallpaper triggers a full theme refresh across all running applications
- **Color Consistency** ‚ÄĒ every UI element (bar, notifications, launcher, terminal) shares the same palette derived from the wallpaper

---

### ‚ö° power-mode

> `/.config/power-mode/`

A custom CLI tool for managing CPU performance profiles on-the-fly ‚ÄĒ useful for switching between battery-saving and full-performance modes without touching system settings manually.

- **Presets** ‚ÄĒ `ultimate` (full power) and `balanced` (power saver) quick-apply modes
- **Fine-Grained Control** ‚ÄĒ `-min` / `-max` frequency bounds and `-gov` governor selection (`powersave` / `performance`)
- **TLP Integration** ‚ÄĒ coordinates with TLP (`/.config/tlp/`) for battery management policies
- **Zsh Completion** ‚ÄĒ full autocompletion with contextual hints registered via `compdef`

---
